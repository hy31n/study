WEEK3

SSL/TLS

SSL과 TLS 모두 서버, 애플리케이션, 사용자 및 시스템 간의 데이터를 암호화하는 통신 프로토콜이다. 네트워크를 통해 연결된 두 당사자를 인증하므로 데이터를 안전하게 교환할 수 있다.
TLS는 암호화 및 인증을 지원하는 보안 통신 프로토콜이다. SSL이 더 이상 사용되지 않기 전에는 SSL이 이 역할을 했습니다. TLS와 SSL 모두 핸드셰이크 프로세스를 용이하게 하고 브라우저와 웹 서버 간에 암호화된 통신을 설정하는 디지털 인증서를 사용한다.

리버스 프록시

리버스 프록시로 웹 서버를 설정할 경우 사용자가 example.com 웹 서비스에 데이타를 요청하면 Reverse Proxy 는 이 요청을 받아서 내부 서버(보통 WAS 입니다)에서 데이타를 받은후에 이 데이타를 사용자에게 다시 전달한다.
대부분의 WAS는 Web Server 기능을 제공하므로 리버스 프록시 없이 내부 WAS가 직접 서비스를 제공해도 되지만 이렇게 구성하는 이유가 있다. 웹 서버가 해킹당해도 2차 방화벽을 다시 뚫어야 하므로 보안 강화에 좋다.

블록태그와 인라인태그

블록 태그 : 내용물의 크기와 상관없이 너비는 항상 100%를 차지한다
블록 태그 종류
div : 블록 컨테이너를 나타내기 위한 의미없는 태그
p : 문단
form : 사용자가 입력한 데이터를 서버로 전송하기 위해 사용한다.

인라인 태그 : 좌우 공간을 필요한 만큼만 차지한다.
인라인 태그 종류
span : 인라인 요소들을 그룹화하거나 영역을 나눌때 사용
a : 링크를 통해 다른페이지, 문서 등을 연결해주는 태그
button : 클릭할 수 있는 버튼을 생성하는 태그
input : 사용자로부터 입력을 받을 때 사용하며 form태그 내부에 많이 쓴다.

flex

레이아웃 배치 전용 기능으로 컨테이너에 적용하는 속성, 아이템에 적용하는 속성이 있다.
display: flex; : 아이템들이 가로 방향으로 배치되고, 자신이 가진 내용물의 width 만틈만 차지하게 된다. height는 컨테이너의 높이만큼 늘어난다.

grid

grid는 두 방향(가로-세로) 레이아웃 시스템이다.
그리드 컨테이너 (Grid Container)
display: grid를 적용하는, Grid의 전체 영역입니다. Grid 컨테이너 안의 요소들이 Grid 규칙의 영향을 받아 정렬된다고 생각하면 된다.
그리드 아이템 (Grid Item)
Grid 컨테이너의 자식 요소들입니다. 바로 이 아이템들이 Grid 규칙에 의해 배치된다.
